## GROUND RULES (read before doing anything)
- Do NOT create new services, providers, or packages unless explicitly told to.
- Use only what already exists in `lib/services/`, `lib/providers/`, and `lib/data/database/`.
- Prefer `ref.watch` over `ref.read` in widget builds. Use `ref.read` only inside callbacks.
- Every async operation must have a loading state and an error state in the UI.
- Do not rename or move files. Only modify file contents.
- After each section, list every file you modified.
